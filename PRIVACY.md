# Privacy Policy

**Last updated:** January 2026

This privacy policy describes how FraudShield AI ("we", "us", "our") collects, uses, and protects your data.

---

## What Data We Collect

### Transaction Data

When you submit transactions for analysis, we collect:

| Data | Purpose | Retention |
|------|---------|-----------|
| Amount | Risk scoring | 90 days |
| Payee name | Pattern matching | 90 days |
| Reference | Keyword analysis | 90 days |
| Timestamp | Timing analysis | 90 days |
| Source | Analytics | 90 days |

### Usage Data

We collect anonymized usage data:

- API request counts (no payloads)
- Feature usage statistics
- Error rates and types
- Response times

### Account Data (Future)

When accounts are implemented:

- Email address
- Organization name
- API keys (hashed)

---

## How We Use Your Data

### Primary Use

- **Fraud Analysis** — Analyze transactions for risk indicators
- **Explanation Generation** — Create human-readable risk explanations
- **Pattern Matching** — Compare against known fraud patterns

### Secondary Use

- **Service Improvement** — Aggregate patterns to improve detection
- **Performance Monitoring** — Ensure system reliability
- **Security** — Detect and prevent abuse

### What We Don't Do

- ❌ Sell your data to third parties
- ❌ Use data for advertising
- ❌ Share individual transaction data externally
- ❌ Train AI models on your specific data without consent

## Your Rights

Under UK GDPR, you have the right to:

| Right | Description | How to Exercise |
|-------|-------------|-----------------|
| **Access** | Get a copy of your data | Email privacy@fraudshield.ai |
| **Rectification** | Correct inaccurate data | Email with corrections |
| **Erasure** | Delete your data | Email deletion request |
| **Portability** | Export your data | Request JSON export |
| **Objection** | Object to processing | Email with objection |
| **Restriction** | Limit processing | Email with request |

We respond to requests within 30 days.

| Data Type | Retention Period | Deletion Method |
|-----------|------------------|-----------------|

## Data Security

We protect your data with:

- **Encryption in transit** — TLS 1.3 for all connections
- **Encryption at rest** — AES-256 for stored data
- **Access controls** — Role-based access to systems
- **Audit logging** — All data access logged
- **Regular audits** — Security reviews and penetration testing

See [SECURITY.md](SECURITY.md) for full security documentation.
